Trivia hunt is a scalable quiz application built using a microservices architecture that enables multiple players to participate in live quiz sessions simultaneously. The system consists of six independent services—API Gateway, Player Service, Quiz Service, Room Service, Leaderboard Service, and Eureka Registry—allowing each component to scale and evolve independently.
The platform leverages WebSocket-based bidirectional communication to deliver quiz questions, player responses, score updates, and game events in real time, ensuring a synchronized experience for all participants with minimal latency. Redis is used as an in-memory datastore to manage active sessions, maintain game room state, and track live scores, providing extremely fast read and write operations during gameplay.
Service discovery is handled through Eureka, while Spring Cloud Gateway provides centralized routing and load balancing across all microservices. The entire application is containerized using Docker and orchestrated with Docker Compose, enabling seamless deployment of all services, including Redis and the service registry, through a single command. The architecture is designed for scalability, reliability, and high-performance real-time interactions, making it well-suited for concurrent multiplayer environments.
